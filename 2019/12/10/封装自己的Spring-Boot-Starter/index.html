封装自己的Spring Boot Starter

发表于 2019-12-10 | 分类于 Spring ， Spring Boot

Starter

我们在使用Spring Boot的时候常常要引入一些Starter，例如spring-boot-starter-web，官方为我们提供了几乎所有的默认配置，很好的降低了使用框架时的复杂度，所以在用xxx-starter的时候，可以不用费心去写一些繁琐的配置文件，即使必要的配置在application.properties或application.yml中配置就可以了，当你实现了一个Starter，可以在不同的项目中复用，非常方便，今天我们来编写自己的Starter。

命名

官方提供Starter的命名规则spring-boot-starter-xxx

非官方Starter的命名规则建议为xxx-spring-boot-starter

封装自己的Starter

新建maven项目

新建maven项目，设置pom.xml文件，增加依赖

spring-boot-configuration-processor不是必须的，它的作用是编译时生成spring-configuration-metadata.json，此文件主要给IDE使用。配置此JAR相关配置属性在application.properties中，你可以用Ctrl+鼠标左键点击属性名，IDE会跳转到你配置此属性的类中。并且编写application.properties会有代码提示。

创建属性配置类

创建一个ConfigurationProperties用于保存你的配置信息，如果你的项目不使用配置信息则可以跳过这一步，不过这种情况非常少见

创建自动配置类

创建一个AutoConfiguration，引用定义好的配置信息。该类主要用于创建核心业务类实例。在AutoConfiguration中实现所有starter应该完成的操作。

二选一即可，区别在于，注解方式需要显式的在启动类上开启@EnableXdfsStarter注解
