动态修改Logback日志级别的两种实现

发表于 2019-03-20 | 分类于 JAVA ， Logback

前言

线上系统出现问题时，第一反应是查看日志，但是为了效率，经常会关闭debug级别日志，这时候通常的做法是联系运维人员，手动修改日志的级别，然后重启服务；那么有没有一种办法不用重启服务也可以修改日志级别，问题排查解决后，再把日志级别修改回来？当然是有的，本文就是动态修改logback日志级别的两种实现。

实现

配置实现

当前实现需要手动修改配置文件，但是无需重启服务，logback会监控文件变化动态修改日志级别

完整logback.xml

代码实现

当前实现，无需运维人员修改配置文件，只需对外暴露接口，调用时传入日志级别即可

LoggerContext loggerContext = (LoggerContext) LoggerFactory.getILoggerFactory();
loggerContext.getLogger("com.hsuns").setLevel(Level.DEBUG);
