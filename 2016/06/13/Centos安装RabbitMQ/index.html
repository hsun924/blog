发表于 2016-06-13 | 更新于 2016-08-17 | 分类于 RabbitMQ

安装Erlang

RabbitMQ是基于Erlang的，所以首先必须配置Erlang环境。
从Erlang的官网http://www.erlang.org/download.html下载最新的erlang安装包,我下载的版本是otp_src_18.3.tar.gz

下载后解压
进入otp_src_18.3执行以下命令 (不用Java编译，故去掉java避免错误)

出现如下错误

根据错误提示，缺少gcc
安装gcc
yum -y install gcc

yum 安装g++时，用yum list的程序名不是 g++，而是 gcc-c++
yum list gcc-c++

yum -y install gcc-c++.x86_64


安装make
yum -y install make

yum -y install perl


yum -y install ncurses-devel

安装缺失依赖
yum -y install openssl-devel
yum -y install unixODBC
yum -y install unixODBC-devel

编译

安装完成后进入/opt/erlang/bin，执行./erl测试是否安装成功

增加环境变量

source profile使得文件生效

安装RabbitMQ

下载


rpm –nodeps -Uvh rabbitmq-server-3.6.2-1.noarch.rpm

/usr/lib/rabbitmq/bin/rabbitmq-server start

管理界面

./rabbitmq-plugins list
./rabbitmq-plugins enable rabbitmq_management

默认guest/guest
只能本地访问

./rabbitmqctl add_user hsun hsun
./rabbitmqctl set_user_tags hsun administrator
./rabbitmqctl set_permissions -p / hsun “.“ “.“ “.*”
